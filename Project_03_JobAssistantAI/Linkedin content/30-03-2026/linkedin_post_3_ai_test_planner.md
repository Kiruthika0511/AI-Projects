# LinkedIn Post 3 — Topic: I Built My Own AI Test Plan Generator (Live on Vercel)

---

I built my first AI agent. It is not impressive. It works.

And I learned more from building it than from reading about AI for six months.

That surprised me.

---

I am learning AI agents for QA with Pramod. Not just watching lectures — building things. This test plan generator is one of those builds. The kind of project where the learning happens in the doing, not the watching.

The result is a test plan generator. You connect your own LLM API key, connect your Jira account, then enter a ticket ID. It fetches the requirement automatically and returns a structured test plan with scenarios and coverage areas. No copy-pasting. No context switching. Just a ticket number.

I deployed it to Vercel last week: https://ai-testing-zeta.vercel.app/

It is a first working version, not a polished product. The output quality depends on the model you bring — GPT-4o gives noticeably sharper results than smaller models. That was a lesson in itself: the agent is only as good as the model behind it.

But here is the part that stuck with me.

The moment it generated a test plan I had not thought of — for a requirement I wrote myself — I understood AI differently. Not from a theory. From my own tool, running live.

Learning AI by building is genuinely different from learning AI by reading about it.

If you have been waiting until you know enough to start — you are waiting for the wrong thing.

Start messy. Ship the first version. Learn from what breaks.
